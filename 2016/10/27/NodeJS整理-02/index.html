nodeJS核心模块fs和http中路径问题

nodeJS核心模块fs

var fs = require('fs'); 通过fs这个模块来实例化一个fs对象，使用这个对象对文件进行操作

fs内置API

  • readFile(path,[option],callback);
  • path:指定要读取文件的路径
  • options:可选参数，指定要以何种编码格式，来读取文件内容，默认是null，一般文件的编码格式都是utf-8;
  • callback:回调函数，指定当文件读取成功后，如何操作读取到的内容,这个回调提供两个参数 err data
  • 如果没有发生错误的话，第一个参数Error对象为null,第二个参数是读取到的内容
  • 如果读取文件失败，第一个参数是一个真正的Error对象,第二个参数是undefined
  • writeFile(path,data,[options],callback)
  • path:要写入文件的路径 (如果该路径下文件不存在，则会默认创建这个文件)
  • data:要写入的数据内容
  • options：可选参数，写入的数据内容的编码格式，此参数默认是utf-8，所以这个参数一般可以不写
  • callbackl:回调函数，这个回调函数只有一个参数err， 如果写入成功，err是null 如果写入不成功 err是个Error对象

​ fs.writeFile这个api之后写入的内容会把之前这个文件中的内容覆盖掉。

  • appendFile(path,data,callback)

  • path: 文件路径

  • data:向文件中追加内容

  • callback：只有一个err参数 同上，如果追加成功 err是null 如果追加不成功 err是个对象 if判断会成立。

    值得注意的是，如果文件不存在也会默认创建这个文件，然后追加内容到这个空文件中。

fs模块中的路径问题

  • 在使用fs模块操作文件的时候，通过’./‘去查找文件的时候，并不是根据当前代码所在的目录去查找的
  • 而是根据当前运行node命令所在的目录去查找这个’./‘相对路径的。(也就是动态拼接了这个两个路径来寻找文件)
  • 为了解决node中fs模块路径的问题，node提供了一个__dirname
  • 经过测试发现__dirname拿到的永远都是node所运行的js文件所在的根目录，而不再是开启node命令所处的根目录。
  • 所以，今后凡是在fs模块中，需要操作路径时，一定要使用__dirname去拼接这个路径，
  • 最后值得注意的是，使用__dirname拼接路径的时候，拼接的字符串开头不能有./ ../ 只能是 /开头，然后可以决定是往下找还是往上跳
  • 笔者不太建议js文件路径低于需要操作的文件的路径，一般开发中也很少会出现这种情况，如果要往上掉 __dirname+”/../“这样操作

nodeJS核心模块http

  1. http核心模块，也是node官方提供的，能够很方便的创建一个web服务器
  2. var http = require(‘http’);
  3. 创建一个web服务器
  4. var server = http.createServer();
  5. 指定该服务器监听的端口号，再启动服务器
  6. server.listen(3000,()=>{//当服务器启动ok之后，会调用回调函数。console.log(“Sever running at 127.0.0.1：3000”)});
  7. 监听客户端的请求，并处理请求，最后把处理的结果通过网络返回给客户端，通过server.on(‘request’)为服务器绑定请求事件，每当有新的客户端请求我们服务器的时候，都会触发这个事件
  8. 每当有请求过来，都能拿到这次请求的request对象，里面保存着请求头，请求报文，同样也有一个response对象，服务器处理完这个请求之后，可以通过这个对象将一些对应的数据通过网络返回给给我发起请求的页面。
  9. server.on(‘request’,(request,reponse)=>{})
  10. 在上面的这个箭头函数中，通过request.url 可以获取用户请求的URL地址
  11. 比如127.0.0.1:3000/index.html就比如这个请求，那么获取到的URL是 ‘/index.html’
  12. 根据用户请求的URL，返回对应的数据，
  13. 如果返回的数据是比较简单的 比如文字或者单独的标签，那么需要指定返回数据的编码格式，不然返回给前台的数据将会是乱码
  14. response.writeHeader(200,{“Content-Type”:”text/plain;charset=utf-8”});
  15. 如果我需要返回一个完整的页面给用户 那么不需要指定编码格式了，因为在返回的页面标签中已经指定了编码格式
  16. 后台读取数据，比如读取html文件，使用fs.readFile(__dirname+’相对于当前app.js的路径’，(err,data)=>{ })app.js一般放在项目的根目录下
  17. 一般情况下，我们返回给用户的数据是引入了其他资源的 这里以html为例，我需要返回一个index.html的页面内容给向我发起请求的用户，
  18. index.html页面引入了css、js资源，那么根据nodeJS请求这些静态资源的规则，默认是将根目录路径css、js的 href src属性进行拼接。基于这个规则设定，我们在index.html页面中引入这些静态资源的时候 都可以直接以项目根路径为基准，向下找。
  19. 通过以上的设置，那么在请求静态资源的时候，获取到的URL值直接就是css、js这些文件的href、src的路径，如此就直接可以读取数据并返回给用户了。
  20. 但是对于index.html这些资源而言，获取前台的URL值 跟后台要读取的index.html的路径就不是对应的了，起码会加多一层文件夹名称比如:
  1. 这个时候我们在想，如果用户能够在向后台请求index.html页面的时候，加多一层/views/,那样后台获取到的URL就跟读取后台index.html的路径一致。
  2. 如果这种页面的路径有能够和URL保持一致，再加上index.html中的css、js的路径也跟URL是保持一致的，那么我们读取数据就可以使用获取到的URL的值，使用这个值的当做读取数据的读取路径，这样一下子就可以处理好前台所需的所有数据。
