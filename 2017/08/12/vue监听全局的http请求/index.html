vue监听全局的http请求

发表于 2017-08-12 | 分类于 Vue.js

  1. 现在有一个需求，就是为了更好的用户体验，我们在请求资源的时候，需要给用户一些友情提示信息，但是又不想给每次请求开始和响应结束设置这些提示的东西，这个时候就需要全局设置http请求和响应之间的友情提示。

    Vue.http.interceptors.push(function(request, next) {    
  // modify request
  //request.method = 'POST';
    这里写每次请求开始需要做什么事

  // continue to next interceptor
  next(function(response) {

    // modify response
    //response.body = '...';
    //这里写每次响应成功需要做什么事

  });
});
  • 这个默认的例子的话，是每次请求的时候 都修改该次请求的方式为post
  • 然后每次响应回来的时候，修改响应回来的内容。
  • 然后我们可能会有这样的需求，用一下mint-ui中的Toast 这个时候就可以在请求开始阶段设置出来一个Toast，然后在响应回来之后，close这个Toast。

备注：在main.js中导入mui的js，然后通过接收的变量调用mui提供的方法创建一个mask，这个mask会以一个标签的形式被插入到body中，然后通过审查元素找到这个div的类名，在我们自己的css中 给这个mask定制样式。这个mask的类名是 mui-backdrop。

    import mui from '../src/public/mui/js/mui.min.js';
        var mask = mui.createMask(() => {
            // console.log('createMask')
        });
        //拦截$http发送请求,
        Vue.http.interceptors.push(function(request, next) {
            mask.show(); //请求前要做的
            next(function(response) {
                setTimeout(() => {
                    mask.close();
                }, 500);
            });
});

common.css中修改mask的样式

    .mui-backdrop {
        position: fixed;
        top: 0;
        right: 0;
        bottom: 0;
        left: 0;
        z-index: 1998;
        background-color: rgba(0, 255, 0, 0.2);
    }
